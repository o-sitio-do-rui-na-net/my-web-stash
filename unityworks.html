Procedural generated voxel world with some physics and MLagents and remote control via OSC and Microphone

Context

During a semester in the discipline of IAV -Interação em Ambientes Virtuais(Interaction in Virtual Ambient) with the help of the teacher we developed a procedural generated voxel world.
The final evaluation project was to use our creativity and knowledge acquired from other projects in the discipline to populate this world.

Overview

On that same empty world, I used the knowledge acquired form previous works to convert OSC commands a 3d hoover flight controller that I developed to control a plane as well as trained agents to find and collect food on the map.
The idea was to bombard the map with missiles and bullets shoot from the plane using mouth sounds (Whistle and blows) and causing lots of small cubes to fly with the impact. To do this I had to figure out how to make this controls, animations and physics work with the map generation architecture.

Plane Controller



For the plane controller I had to not only translate OSC commands in a 3d movement vector but also negate gravity (because hoovering is a more user-friendly flight to do and work with) and collision momentum gained after colliding with something in a world with no drag.
To do this I had to play around with Unity physics engine as well as creating a system that in fact moves the plane.
The output of this was a plane that could be controlled by the mobile phone and when we would whistle he would shoot a barrage of bullets and when we blow he would shoot a missile.

With some abstraction and an interface like this, one can actually fly a plane :D

Agent training and deployment


To train the agents I had to replicate the environment that they were going to act in as well associate the output of the neural network to movement on the map.
Essentially this consisted in creating a movement script for the agent that is walking in both directions, front and backwards, and associating each type of movement with discrete actions produced by the neural network. Also, the agents needed vision of the world and for that I gave them the nearest object of interest.
The agents were rewarded each time they were close enough to this object of interest. This train resulted in a collector type agent capable of wonder and collect objects when seen.

Tecnological Components Explored

This academic project was developed on Unity using the MLAgents toolkit as well as an OSC connector for unity.
The code is written in C#.

OSC is protocol that has a wide use the fields of music, robotics, video performance interfaces, and inter-process communication.
Using an unity interface for OSC is possible to associate object behaviours in a world to OSC commands.

To put it simple the MLAgents toolkit provides a Python API for Unity that uses reinforced learning and other types of machine learning to train agents.
